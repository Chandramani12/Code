# Ticket Routing, Access Control & Validation Issues – Complete Requirement Document

---

# 1. IT Help Desk Ticket Routing

	

* User location
* Organization
* Team mapping configuration

### Expected Behavior

* Tickets should only be routed to the matching IT Service team.
* Tickets should not be assigned to other locations or unrelated teams.
* Routing should follow organization-wise and location-wise mapping.

---

# 2. DBA Ticket Routing

When a user creates a ticket related to DBA support, the ticket should be automatically routed only to the DBA team according to:

* User location
* Organization hierarchy
* Team mapping rules

### Expected Behavior

* DBA tickets should only be visible to DBA teams.
* Tickets should not appear for unrelated departments or managers.
* System should validate organization and location before assignment.

---

# 3. Manager Role Access Restriction

When an admin assigns a manager role for the DBA team, that manager should only have access to DBA-related data based on the assigned location and organization.

## DBA Manager Access Should Include

* DBA tickets for the assigned location only
* DBA employees for the assigned location only
* DBA dashboard and modules related to the assigned location and organization

## Expected Behavior

* DBA managers should not be able to view:

  * Other team tickets
  * Other location tickets
  * Other organization data

### Access Should Be Restricted Based On

* Team
* Location
* Organization
* Role permissions

### Current Issue

Currently, DBA managers are unable to correctly view or access DBA team data location-wise according to their assigned permissions.

---

# 4. Team Mapping Error During Ticket Creation

Sometimes, while creating a DBA-related ticket, users receive the following error:

```txt id="7o1xq2"
No team is set up at your location to handle this category.
Ask an admin to create the matching team at your location, or choose a different category.
```

## Current Issue

```txt id="x2gsr9"
POST http://localhost:3008/api/tickets-generate
500 Internal Server Error
```

The system is throwing an Internal Server Error instead of handling the validation properly.

## Expected Behavior

The system should:

* Validate category, organization, and location mapping before ticket creation
* Show a professional validation popup if no matching team exists
* Prevent backend crashes and avoid 500 errors

---

# 5. Professional Validation & Error Messages

If users:

* Enter incorrect values
* Select invalid options
* Submit incomplete forms
* Choose unmapped categories

the system should display professional popup validation messages.

## Recommended UI

* Toast notifications
* Modal alerts
* Validation popups
* Inline form validation messages

## Example Messages

* Invalid category selected
* Team mapping not found
* Required fields missing
* Unauthorized access
* Invalid organization/location mapping

Browser default alerts should not be used.

---

# 6. DBA Manager Ticket Visibility Issue

Currently, when a DBA manager logs into the system, DBA tickets are not visible correctly in the manager dashboard.

## Expected Behavior

DBA managers should only see:

* DBA tickets
* DBA employees
* DBA approvals
* DBA activity logs
* Tickets related to their assigned location and organization

## Current Issue

The filtering and visibility logic is not working correctly for DBA managers.

---

# 7. Incorrect Ticket Visibility for IT Services Manager

Currently, IT Services managers are able to view both:

* IT Help Desk tickets
* DBA tickets

## Expected Behavior

IT Services managers should only have access to:

* IT Help Desk tickets
* IT Services employees
* IT Services dashboards/modules
* Tickets belonging to their assigned location and organization

DBA tickets should not be visible unless explicitly authorized.

Implement strict role-based and team-based access control.

## Access Levels

### Admin

* Full system access
* All organizations
* All locations
* All teams

### DBA Manager

* Only DBA team access
* Assigned organization only
* Assigned location only

### IT Services Manager

* Only IT Services access
* Assigned organization only
* Assigned location only

### Employee/User

* Self-created tickets only
* Assigned tickets only
